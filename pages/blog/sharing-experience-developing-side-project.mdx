# 1. Developing API

Bahasa pemrograman Rust sebenarnya sudah saya pelajari dari tahun kemarin,
dan sudah ada 2 project lainnya yang menggunakan bahasa pemrograman Rust,
penggunaan tidak begitu komplek dan hanya penyimpanan data cuma di file JSON.

Untuk develop API ini saya lebih melihat referensi code dari library Tokio Hyper & Sqlx.
Karena experience untuk menuliskan code dari bahasa pemrograman tersebut saya kurang
dan saya tidak 100% memahami teori dari bahasa pemrogramannya.
Selain itu library tersebut juga sudah menyediakan beberapa contoh aplikasi yang di local saya berhasil dijalankan.

Kalau boleh jujur, saya melakukan beberapa kali penyesuaian skeleton di API ini.
yang saya lakukan untuk mengerjakan aplikasi API ini,

1. Start project dengan include framework Hyper.
2. Develop handler agar request path & request method sesuai dengan domainnya.
   misalnya /accounts kusus untuk account, /transaksi kusus untuk transaksi dan sebagainya.
3. Restruktur skeleton,
   agar request method GET bisa multiple kondisi.
4. Menambahkan library Sqlx agar CRUD bisa terhubung dengan database.
5. Restruktur skeleton lagi,
   agar Sqlx bisa menerapkan rollback transaksi ketika ada kegagalan transaksi di salah satu query ke database.

Yang paling sulit ketika develop aplikasi ini adalah

1. Anda tidak bisa senaknya mendefinisikan variable senaknya,
   every type of variable di bahasa pemrograman ini is matter.
2. Penerapan transaksi di Sqlx tidak mudah.
   karena fitur tidak include dengan librarynya,
   ada solusinya tetapi saya temukan di forum.
   Di aplikasi ini sudah include transaksi cuma untested.
3. Referensi terbatas, contoh penggunaan code di dokumentasi tidak banyak.

Dibalik kesulitan itu,
sebenarnya bahasa pemrograman ini lebih saya sukai daripada PHP & Javascript.
Karena saking strict-nya semakin saya ingin tau cara membuka kunci pintu tersebut.

# 2. Developing Mobile

Sebenarnya saya belum pernah menggunakan Kotlin bahkan Java.
Tetapi saya yakin saya bisa menyelesaikan side project ini.
Yang saya memulainya dengan,

1. Membuat project awal dari android studio dan beberapa refrensi dari android developer.
2. Selanjutnya mempelajari jetpack compose beserta dengan course dari android developer dan
   saya cuma menyelesaikan 3 course dari android developer terkait jetpack compose.
3. Dan langsung mengerjakan layout layout dengan membayangkan seharusnya seperti apa.
4. Menambahkan beberapa library milik Ktor dan membuat request ke API dengan menaruh request di ViewModel.
5. Menambahkan beberapa library untuk Room & Ksp.

Kesulitan yang saya alami ketika develop mobile ini adalah,

1. ternyata untuk implementasi request API belum saya pelajari,
   dari sana saya menemukan ViewModel.
   Dan hampir sama dengan Rust, Kotlin ada library-nya sendiri untuk encode/decode JSON data menjadi Object.
   Dan juga disini saya belum mengetahui & bagaimana menerapkan request pagination ke lazyload.
2. Karena setiap data di provide dengan request ke API,
   tetapi konfigurasi request API ingin saya simpan di room database.
   Dan juga saya lihat untuk request ke room database membutuhkan context,
   sedangkan beberapa composable component juga request ke API melalui ViewModel.
   Akhirnya saya ambil solusi data konfigurasi di provide Base Activity,
   Activity lainnya saya extends ke Base Activity dan data untuk composable component saya provide oleh Activity.
3. Ketika upload ke play store / app store membutuhkan DUNS,
   yang sampai sekarang belum mendapatkan kabar dari hasil request registrasi DUNS.
4. Preview untuk composable component memakan resource yang tinggi,
   akhirnya saya tidak implementasi Preview untuk composable component.

Menurut saya hasil develop aplikasi mobile ini, cukup menarik,
mudah dipelajari kalau menggunakan jetpack compose dari pada menggunakan resource layout.
Tetapi secara hasil output saya kurang tau, karena proses upload ke play store juga masih terkendala.
